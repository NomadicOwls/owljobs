## Phase Goal

**As a** founder, **I want to** probe a list of potential employer career pages automatically and get a ranked list of detected ATS platforms with confidence scores, **so that** I know which employers to activate next without manually checking each career page.

<objective>
Create the `workers/discover` Cloudflare Worker per D-01 through D-04. The Worker exposes a single `POST /probe` endpoint (founder-triggered on-demand), reads all `pending` rows from `public.candidates` (the table created by Plan 01), fetches each career page URL, detects ATS signatures, and writes results back.

This is the COVG-01 deliverable — the actual "ATS auto-discovery script" required by the requirement. Plan 01 only created the data table; this plan builds the Worker that uses it.

Security: `POST /probe` is protected by a `Authorization: Bearer {DISCOVER_SECRET}` header check per the RESEARCH.md security domain. Without this, any party can trigger expensive network probing.

Detection logic (D-04): 8 ATS platforms with 3 confidence tiers:
- 1.0: Exact URL pattern match in fetch response URL or HTML links (e.g. `/boards/` path for Greenhouse)
- 0.8: Domain match in response or linked URLs (e.g. `.recruitee.com`, `.softgarden.io`)
- 0.6: Script or link tag reference in page HTML (e.g. `<script src="...greenhouse.io...">`)

Purpose: COVG-01 — probes employer career pages and writes ranked ATS discovery results to `public.candidates`.

Output:
- `workers/discover/wrangler.toml` — Worker config (name: owljobs-discover, bindings for Supabase secrets)
- `workers/discover/src/index.ts` — POST /probe handler + ATS signature detection
- `workers/discover/test/probe.test.ts` — unit tests for detection logic (Wave 0 file)
</objective>

    ```typescript
    /**
     * workers/discover — ATS Auto-Discovery Worker (COVG-01)
     *
     * POST /probe — Probes all public.candidates rows with status='pending',
     * detects ATS platform signatures, writes results back.
     *
     * Authorization: Bearer {DISCOVER_SECRET} required on all requests.
     */
    import { createClient } from "@supabase/supabase-js";

    interface Env {
      DISCOVER_SECRET: string;
      SUPABASE_URL: string;
      SUPABASE_SERVICE_ROLE_KEY: string;
    }

    interface Candidate {
      id: string;
      name: string;
      careers_url: string;
      status: string;
    }

    interface DetectionResult {
      ats_type: string | null;
      confidence: number;
      status: "detected" | "unknown" | "error";
    }

    // ATS signature patterns — D-04
    // Each entry: { platform, patterns } where patterns are checked against fetched HTML + URLs
    const ATS_SIGNATURES: Array<{ platform: string; patterns: RegExp[] }> = [
      { platform: "workday",        patterns: [/\/wday\//i, /workday\.com/i] },
      { platform: "greenhouse",     patterns: [/boards\.greenhouse\.io/i, /\/boards\//i] },
      { platform: "lever",          patterns: [/jobs\.lever\.co/i] },
      { platform: "smartrecruiters",patterns: [/jobs\.smartrecruiters\.com/i, /smartrecruiters\.com/i] },
      { platform: "recruitee",      patterns: [/\.recruitee\.com/i] },
      { platform: "softgarden",     patterns: [/\.softgarden\.io/i] },
      { platform: "ashby",          patterns: [/jobs\.ashbyhq\.com/i] },
      { platform: "icims",          patterns: [/\.icims\.com/i, /icims\.com/i] },
    ];

    /**
     * Detect ATS from fetched career page content.
     * Returns the first matching ATS with tiered confidence:
     *   1.0 — pattern found in final response URL (exact redirect/URL match)
     *   0.8 — pattern found in page HTML (href/src of a linked domain)
     *   0.6 — pattern found in script/link tag src references
     */
    function detectAts(html: string, finalUrl: string): DetectionResult {
      for (const sig of ATS_SIGNATURES) {
        // Tier 1: URL match (confidence: 1.0)
        for (const p of sig.patterns) {
          if (p.test(finalUrl)) {
            return { ats_type: sig.platform, confidence: 1.0, status: "detected" };
          }
        }
        // Tier 2: href/link in HTML (confidence: 0.8)
        for (const p of sig.patterns) {
          if (p.test(html)) {
            // Distinguish href/src presence (0.8) vs inline text reference (0.6)
            const hrefMatch = new RegExp(`href=["'][^"']*${p.source}[^"']*["']`, "i").test(html) ||
                              new RegExp(`src=["'][^"']*${p.source}[^"']*["']`, "i").test(html);
            return {
              ats_type: sig.platform,
              confidence: hrefMatch ? 0.8 : 0.6,
              status: "detected",
            };
          }
        }
      }
      return { ats_type: null, confidence: 0, status: "unknown" };
    }

    async function probeCandidate(candidate: Candidate): Promise<DetectionResult> {
      try {
        const response = await fetch(candidate.careers_url, {
          redirect: "follow",
          headers: { "User-Agent": "OwlJobs-Discover/1.0 (+https://owljobs.com/about)" },
          signal: AbortSignal.timeout(10_000),
        });
        const html = await response.text();
        return detectAts(html, response.url);
      } catch (_err) {
        return { ats_type: null, confidence: 0, status: "error" };
      }
    }

    export default {
      async fetch(request: Request, env: Env): Promise<Response> {
        // Authorization check — MUST be first
        const auth = request.headers.get("Authorization");
        if (!auth || auth !== `Bearer ${env.DISCOVER_SECRET}`) {
          return new Response("Unauthorized", { status: 401 });
        }

        if (request.method !== "POST" || new URL(request.url).pathname !== "/probe") {
          return new Response("Not Found", { status: 404 });
        }

        const db = createClient(env.SUPABASE_URL, env.SUPABASE_SERVICE_ROLE_KEY, {
          auth: { persistSession: false },
        });

        // Fetch all pending candidates
        const { data: candidates, error } = await db
          .from("candidates")
          .select("id, name, careers_url, status")
          .eq("status", "pending");

        if (error) {
          return new Response(JSON.stringify({ error: error.message }), {
            status: 500,
            headers: { "Content-Type": "application/json" },
          });
        }

        const results: Array<{ id: string; name: string } & DetectionResult> = [];

        for (const candidate of (candidates ?? []) as Candidate[]) {
          const result = await probeCandidate(candidate);
          // Write result back to candidates table
          await db
            .from("candidates")
            .update({
              ats_type: result.ats_type,
              confidence: result.confidence,
              probed_at: new Date().toISOString(),
              status: result.status,
            })
            .eq("id", candidate.id);
          results.push({ id: candidate.id, name: candidate.name, ...result });
        }

        return new Response(
          JSON.stringify({ probed: results.length, results }, null, 2),
          { status: 200, headers: { "Content-Type": "application/json" } },
        );
      },
    };
    ```

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Founder → POST /probe | Must present DISCOVER_SECRET; unauthenticated requests return 401 |
| POST /probe → external careers pages | Worker fetches arbitrary URLs from candidates table; responses are untrusted HTML |
| Worker → Supabase public.candidates | Service-role key bypasses RLS — key must be a Worker secret, never in source |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Spoofing | POST /probe authentication | mitigate | `Authorization: Bearer {DISCOVER_SECRET}` check is the first statement in the fetch handler; any non-matching request returns 401 immediately |
| T-02-10 | Tampering | careers_url values in candidates table | accept | Only founder can insert into candidates (service-role only, no public insert policy); SSRF risk is accepted (founder controls the URL list) |
| T-02-11 | Information Disclosure | SUPABASE_SERVICE_ROLE_KEY | mitigate | Stored as Worker secret via `wrangler secret put`; never in source or wrangler.toml |
| T-02-12 | Denial of Service | Unbounded fetch loop on large candidates table | mitigate | `AbortSignal.timeout(10_000)` per probe; Worker has 30s CPU cap; large batches handled by sequential processing (not Promise.all) |

</threat_model>

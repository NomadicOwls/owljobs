Fix WR-05 (SEO-01 gap): `[slug].astro:67` uses `(job.job_sources ?? []).some((s) => aggregatorSources.has(s.source))` to detect aggregator jobs. This check fires true if ANY source row references Adzuna or JSearch — including native ATS jobs (Workday, Greenhouse, SuccessFactors, etc.) that also appear in aggregator search results. A Vestas job enriched with a full description incorrectly loses its JSON-LD rich result.

The fix is a single-line change:

```typescript
// BEFORE (broken — fires on any cross-listed source):
const isAggregator = (job.job_sources ?? []).some((s) => aggregatorSources.has(s.source));

// AFTER (correct — checks the employer's ATS type):
const isAggregator = ["adzuna", "jsearch"].includes(job.employers?.ats_type ?? "");
```

After the fix, the `aggregatorSources` Set is no longer used (remove it). A new test block in `apps/web/test/jobs.test.ts` asserts the correct pattern.

Fix CR-01: Four operational debug endpoints in `workers/ingest/src/index.ts` — `/classify-now`, `/ingest-now`, `/reclassify-ambiguous`, `/enrich-now` — have no authentication. Any HTTP client with knowledge of the Worker URL can trigger Adzuna/JSearch API calls (paid quota), Workers AI inference costs (classify), Google Indexing API pings (200/day budget), and Cloudflare Pages rebuilds.

Add a `Bearer ${env.INGEST_SECRET}` check on the four debug endpoints. `/jobs.json` is the public production endpoint and MUST remain unauthenticated.

The pattern follows `workers/discover/src/index.ts:96-99` which uses an identical per-handler Bearer check:
```typescript
const auth = request.headers.get("Authorization");
if (!auth || auth !== `Bearer ${env.DISCOVER_SECRET}`) {
  return new Response("Unauthorized", { status: 401 });
}
```

Purpose: Security hardening — prevents unauthorized callers from burning paid API quota or triggering expensive operations. Required before deploying workers/ingest to production with live credentials.

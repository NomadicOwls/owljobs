## Phase Goal

**As a** wind turbine technician using Google search, **I want to** find newly-posted OwlJobs roles in Google search results within 48 hours of posting, **so that** I never miss time-sensitive openings.

<objective>
Extend the existing Google Indexing API integration (currently fires on EXPIRY only — Phase 1) to also fire on (a) job CREATION (in `ingest.ts` after `upsertJob` inserts a new row) and (b) DESCRIPTION UPDATE (in `enrich.ts` after writing `description`).

Critical correctness fixes from research:
- **Pitfall 8**: pings MUST use `buildPublicUrl(niche, job.id)` (the owljobs.com URL Google can crawl), NOT `job.canonical_url` (the employer's ATS URL). Phase 1's expire.ts uses `canonical_url` — this is a known bug; we do NOT fix it here, but Phase 2 must establish the correct pattern.
- **Pitfall 2**: Indexing API has a 200/day quota; expire.ts already caps at `PING_BUDGET_PER_RUN = 100`. We add `CREATION_PING_BUDGET = 50` and `DESCRIPTION_PING_BUDGET = 50` so total budget across all ping types stays within quota.

Purpose: SEO-03 — job creation, expiry, and description updates ping the Google Indexing API; URLs appear in Search Console within 48 hours.

Output:
- `workers/ingest/src/ingest.ts` — add `buildPublicUrl` helper + creation ping after each `upsertJob` insert (in all 5 existing ingest functions: ingestWorkday, ingestGreenhouse, ingestSuccessFactors, ingestRecruitee, ingestSoftgarden)
- `workers/ingest/src/enrich.ts` — add description-update ping after each successful `db.update({ description })`
- `workers/ingest/test/creation-ping.test.ts` — source-contract test asserting the pattern
- `workers/ingest/test/description-ping.test.ts` — source-contract test asserting the pattern
</objective>
